学习使用Mac OS X操作系统，为从Linux桌面环境向Mac环境过渡做好准备！

1. 今天安装了Mac OS X Lion 10.7.1操作系统，安装的过程中碰到了一些问题，具体的是如何设置VirutalBox的问题，具体操作过程如下：
1）new vm，寻则Mac OS X 64位，一路确定，设置磁盘大小的时候，选择20GB，不要太小，太小的话，可能会安装失败。
2）然后选中虚拟机，“Mac OS X”，点击settings，然后修改详细的配置信息，具体有：
x 将cpu的数量修改为2，我的是i5 2520，双核的
x 将最大可用内存的数量调整为1496mb，太大了超过host的内存，vbox会提示警告，主要是会影响host的稳定性
x 将显存数量调整为最大，128m，显存过小的话，根本无法安装。同时激活3d加速，2d加速不激活
x 关闭uefi支持
x 设置storage设备，主要就是将我们待安装的系统对应的iso文件挂载到cd/dvd上。
3）点击开始，进入安装过程，安装过程中appid，可以填也可以不填，最好是到官网注册一个，后面手动下载文件的时候会用到
4）安装完成之后，点击启动，准备启动系统，此时需要注意的是。我们安装后的虚拟机，在启动的过程中还是依赖系统安装盘中的引导程序来引导的，需要在点了启动之后，按F8键，这样可以调出引导菜单，然后选择我们刚安装过的系统，不要再进入iso引导的安装过程。如果把安装程序iso文件卸载掉的话，安装后的mac系统无法正常启动。
这样，系统启动过程中会引导用户进行某些设置，比如账户等等。最后会进入系统登录界面，输入用户名、密码即可登录。


修改系统引导
1) 安装multibeast，通过它来引导系统的启动，让系统不再依赖于之前的引导安装文件。
2) 需要注意的是，安装的时候，一定要把NullPowerManagement给勾上，否则苹果的驱动会代替intel的电源管理驱动，会造成内核panic。
3）安装完成之后，重新启动系统，在进入引导界面时，可以按空格键，此时可以键入内核参数，例如boot: "Graphics Mode"="1600x900x32"，这里设置分辨率是无效的，我们好后解释。如何解决这个问题。
至此，引导的问题解决了。

屏幕分辨率
1）默认的屏幕分辨率在vbox里面是1024x768,这太小了，远远达不到我们的要求。
2）我们要将其设置成1600x900x32，怎么设置呢。分两步进行：
x 首先关闭虚拟机，关闭vbox
x 然后执行vmmanager setextradata "vm naem" "CustomVideoMode1" "1600x900x32"，
执行这行命令，然后重启vbox，重启mac os x虚拟机，分辨率问题就解决了。

编译工具链
安装mac毕竟不是拿来玩耍的，是要学习的，编译工具链还是不能少，但是不管安装什么，gcc总是少不了的，很多东西，甚至是编译器都是由它来构建。
但是mac os x里面没有apt、yum，但是有大神开发了homebrew，我们就用这个来安装。
1）安装homebrew，其官方repo在github上，执行ruby命令安装brew（mac中自带ruby）
2）注意，ruby执行命令安装homebrew的时候，实际上需要穿墙，否则下载容易失败
3）安装了homebrew之后，这个时候brew可依运行了，但是由于缺少建立工具，如编译器等等，无法对下载下来的程序进行编译处理，此时需要安装xcode及其命令行工具
4）从app store中搜索到了xcode，但是版本太新，当前mac os x版本无法运行，所以注册apple developer然后到其官方站点中搜索合适的xcode版本，我下载的时xcode 4.3.2。文件有1.8个G，下载时间比较长。
5) 安装的时候提示当前系统版本为10.7，xcode要求系统版本为10.7.3+，此时怎么办，更新系统呗，更新到了10.7.5.
这里顺便说一下mac os x的版本编号问题，uname -r，根据内核的输出信息，去匹配对应的mac os x发行版。
更新完系统之后，再安装xcode，就成功了。
6）安装xcode command line tools，这个如果通过xcode->preferences->Downloads进行安装的话，由于证书问题，会导致安装失败，此时可以到官网手动搜索合适的版本，然后下载安装。只要能兼容xcode 4.3.2就没有问题。
7）安装成功之后，这样基本的工具链就有了，虽然可能还不像linux下面gnu build collections一样完善，但是至少brew可以跑起来了，可以安装程序了。

快捷键定制
1）修饰键，ctrl、command、options（alt）、capslock
2）根据个人使用习惯，对快捷键进行修改
3）对于ctrl-c、ctrl-v、ctrl-z、ctrl-x等习以为常的快捷键，切换到command按键组合实在是不方便。对其进行修改，只要添加Applications的子项就可以了。每个子项对应着一个应用程序菜单，其名称即菜单的名称，例如Copy、Paste等等。
4）home、end，mac笔记本里面好像没有这两个按键，通过功能键Fn+方向键(左右箭头）来代替
